### Accent Reserved For

The `--color-accent` (#7a5c12) is reserved for exactly these elements in Phase 006 — never applied to decorative or non-interactive surfaces:

1. "Analyze with AI" CTA button background (variant: accent) and its hover state
2. "Accept" button text on tag suggestion cards (solid confidence signal)
3. Progress bar fill during analysis (matches Phase 004 BatchEmbeddingBar pattern)
4. Analysis completion indicator — accent dot or small accent border-left on section
5. Confidence score badge when >80% — accent-toned badge (matches SimilarityScore pattern from Phase 005)
6. Analysis history latest/active entry — accent-subtle background + accent left border bar
7. Focus ring on tag suggestion cards (focus-visible: ring-1 ring-accent/20)
8. Link text in empty state CTA ("Analyze with AI" / "AI 分析")

## Interaction Contracts

### AI-01: AI-Generated Image Description

**Location:** Within AiAnalysisSection in DetailPanel. First sub-section after the section label.

**Content:**
```
SectionLabel: "AI Analysis" (10px, uppercase, text-faint, tracking-0.1em)
Description heading: none (the description is self-evident)
Description text: prose paragraph, DM Sans 14px, text-secondary, line-height 1.6
```

**States:**
1. **No analysis performed (empty):** Description area hidden. Section shows empty state (see AI-04 empty state contract).
2. **Analysis in progress:** Skeleton lines (2-3 lines, animate-pulse, bg-bg) matching description text width.
3. **Analysis complete:** Full description text rendered. Matches mock data pattern — a 1-3 sentence natural language description of the image content.
4. **Analysis error:** Section shows error text (see Error States section). Description area hidden.

**Mock description examples (hardcoded in API stub):**
- "A golden sunset casting warm light over mountain ridges, with silhouetted pine trees in the foreground"
- "A minimalist architectural interior with geometric shadows falling across polished concrete floors"
- "A serene forest stream winding through moss-covered rocks under a canopy of birch trees"

**Interaction:** Read-only, non-interactive. Text is informational only.

### AI-02: Tag Suggestion Cards

**Location:** Within AiAnalysisSection in DetailPanel. Second sub-section, below description, above content analysis.

**Layout:**
```
[Sub-section heading: "Suggested Tags"]  -- 10px Label, uppercase, text-faint
[Tag 1 card: label + 87% + [Accept] [Reject]]
[Tag 2 card: label + 72% + [Accept] [Reject]]
[Tag 3 card: label + 45% + [Accept] [Reject]]
... (typically 3-6 suggestions)
```

**Tag suggestion card structure:**
```
┌────────────────────────────────────────┐
│  "sunset"  ▎████████░░  87%  Accept  拒绝 │
└────────────────────────────────────────┘
   ^-- tag    ^-- confidence bar   ^-- actions
   label      (mini progress bar)  (text-only buttons)
```

**Card styling:**
- Width: full section width (288px within DetailPanel's 320px minus 16px padding each side)
- Background: `bg-surface`
- Border: `border border-border`
- Border radius: `rounded-[2px]`
- Padding: 8px 12px (internal)
- Gap between cards: 4px (xs spacing)
- Hover: `bg-accent-subtle border-accent/20`, 200ms ease-out

**Confidence score bar (mini progress bar within card):**
- Height: 4px (h-1)
- Border radius: `rounded-[2px]`
- Track: `bg-bg` (full width)
- Fill: dynamic width based on score percentage
- Transitions: 200ms ease-out on fill width
- Label: Mono 11px, right-aligned, format: "87%"

**Confidence score color coding (matches Phase 005 SimilarityScore pattern):**

| Score Range | Fill Color | Text Color | Background |
|-------------|-----------|------------|------------|
| 80-100% | accent | accent | accent-subtle (card bg) |
| 50-79% | text-secondary (30% opacity as fill) | text-secondary | transparent |
| 0-49% | text-muted (20% opacity as fill) | text-muted | transparent |

**Accept/Reject buttons:**
- Display: text-only, no background, no border
- Font: DM Sans 12px
- Accept: `text-success` idle, `text-success` underline on hover
- Reject: `text-text-muted` idle, `text-danger` on hover
- Gap between buttons: 8px
- Each button: clickable, 200ms ease-out transition on color
- Layout: right side of card, flex row

**Button labels:**
- Accept: "Accept" / "接受"
- Reject: "Reject" / "拒绝"

**State transitions:**

1. **Idle (all tags visible, no action taken):**
   - All cards render as described above
   - Accept/Reject buttons visible and interactive

2. **Accept clicked on a tag:**
   - Card transitions: background shifts to success at 5% opacity, border to success/20
   - Accept text becomes solid success (bold? no — keep weight consistent, just color change)
   - Reject button fades (opacity 0 after transition)
   - Card remains visible with accepted state for 1.5s, then collapses with 200ms ease-out slide-up
   - Accepted tag is added to the image's existing tags (via appStore) — appears in Tags section above
   - Confetti-free, quiet transition — matches 古卷·灯火克制 aesthetic

3. **Reject clicked on a tag:**
   - Card collapses immediately: fade-out + slide-up, 200ms ease-out
   - No color change — simple removal
   - Rejected tag is discarded (never added to image tags)

4. **All tags accepted/rejected:**
   - Sub-section shows: "All suggestions reviewed" / "所有建议已审阅" in text-faint, 12px
   - Remains visible to indicate the interaction is complete

**Keyboard accessibility:**
- Tab between tag suggestion cards (each card is focusable)
- Enter/Space on focused card; then arrow keys between Accept/Reject
- Focus visible: `ring-1 ring-accent/20` on the card

### AI-04: "Analyze with AI" Trigger Button

**Location:** Within AiAnalysisSection in DetailPanel. Rendered at the top of the section (below section label) when no analysis exists, OR as a small "Re-analyze" / "重新分析" link at the bottom of the section when analysis results exist.

**States:**

| State | Button Appearance | Text |
|-------|-------------------|------|
| Idle (no analysis yet) | Full CTA button, accent background, surface text, 4px radius, 12px font | "Analyze with AI" / "AI 分析" |
| Analyzing | Button replaced with progress bar + text | "Analyzing..." / "分析中..." |
| Complete (results visible) | Small text link at bottom of section | "Re-analyze" / "重新分析" |
| Error | Button resets to idle state + error text above | "Analyze with AI" / "AI 分析" (retry) |

**Idle button styling:**
- Width: auto (content-sized, min 120px)
- Height: 32px (h-8)
- Background: `bg-accent`
- Text: `text-surface` (or appropriate light-on-dark), DM Sans 12px
- Border radius: `rounded-[4px]` — 方章 pattern
- Hover: `bg-accent-hover`, 200ms ease-out
- Focus visible: `ring-1 ring-accent/20`
- Margin: 8px below section label, 12px above description/empty content

**Analyzing state:**
```
[Progress bar: accent fill, 4px height, 2px radius]  80%
"Analyzing..."                                       -- 12px, text-muted, centered
```
- Progress bar replaces button
- Width: full section width (288px)
- Height: 4px (h-1)
- Track: accent at 20% opacity
- Fill: accent, dynamic width, 200ms ease-out transition
- Progress text: DM Sans 12px, text-muted, right-aligned below bar
- Mock progress: increments from 0% to 100% over 1.2-2.8s (configurable delay in API stub)
- Cancel action: none (analysis is non-interruptible for simplicity — matches single-image scope)

**Post-analysis state:**
- Results (description, tags, content) appear with 200ms ease-out fade-in
- Button transforms to small text link: "Re-analyze" / "重新分析"
- Text link styling: DM Sans 12px, text-muted, no underline idle, accent underline on hover
- Positioned at bottom of section, right-aligned

**Interaction:**
- Click triggers `analyzeImage` API stub via `useAiAnalysisStore`
- Button becomes disabled during analysis (prevents double-trigger)
- No confirmation dialog — single click to trigger
- Results render inline within the section (no navigation, no modal)

## Interaction Summary (per AI requirement)

### AI-01: AI Image Description
- **Trigger:** Automatically rendered when analysis results exist
- **Display:** DM Sans 14px prose paragraph, text-secondary, line-height 1.6
- **States:** empty (hidden), loading (skeleton), complete (text), error (hidden)
- **Interaction:** Read-only
- **Keyboard:** Not focusable

### AI-02: Tag Suggestion Cards
- **Trigger:** Automatically rendered when analysis results include tag suggestions
- **Display:** Cards with tag label + confidence bar + accept/reject text buttons
- **States:** idle, hover, accepting, accepted, rejecting, rejected, all-reviewed
- **Interaction:** Click Accept/Reject per card. Focusable via Tab. Arrow keys within focused card.
- **Accepted tag:** Added to image's tag list via appStore. Card transitions to success state then collapses.
- **Rejected tag:** Card collapses immediately. Tag discarded.

### AI-03: Content Analysis
- **Trigger:** Automatically rendered when analysis results include content analysis
- **Display:** Object chips, color swatches with hex, composition prose
- **States:** loading (skeleton), complete (content visible)
- **Interaction:** Read-only. Not focusable.
- **Partial data:** Any sub-section can be independently hidden if its data is absent.

### AI-04: Analysis Trigger
- **Trigger:** User clicks "Analyze with AI" button
- **Display:** Full accent CTA button (idle) → progress bar + "Analyzing..." text (analyzing) → small "Re-analyze" link (complete)
- **States:** idle, analyzing (progress), complete (results visible), error (reset to idle + error text)
- **Interaction:** Single click, no confirmation. Disabled during analysis. No cancel action.

### AI-05: Analysis History
- **Trigger:** Automatically rendered when >0 past analysis runs exist
- **Display:** Chronological list with timestamp + summary + stats
- **States:** loading (skeleton), populated (entries visible), empty (section hidden)
- **Interaction:** Informational only in v0.2. Hover highlights entry. Not focusable.
