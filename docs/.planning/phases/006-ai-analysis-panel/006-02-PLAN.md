  <name>Task 1: Create TagSuggestionCard Component</name>
  <files>src/components/TagSuggestionCard.tsx</files>
  <action>
Create `src/components/TagSuggestionCard.tsx` following the project component pattern: named export, TypeScript interface for props, `cn()` for class merging, 200ms ease-out transitions, no icons, pure text labels.

**Props interface:**
```typescript
interface TagSuggestionCardProps {
  imageId: string
  tag: TagSuggestion
}
```
Import `TagSuggestion` from `@/lib/api/analysis`. Import `useAiAnalysisStore` from `@/stores/ai-analysis-store`. Import `useTranslation` from `@/lib/i18n`.

**Component has these internal states (derived from store):**
1. **Idle** — neither accepted nor rejected.
2. **Accepting** — user just clicked Accept, transitioning to accepted.
3. **Accepted** — tag is in store's acceptedTags Set. After 1.5s timeout, card visually collapses (but state remains — controlled by a local `visible` boolean).
4. **Rejected** — tag is in store's rejectedTags Set. Card immediately transitions to collapsed.

**Rendering logic:**

Read from store:
- `isAccepted = isTagAccepted(imageId, tag.label)`
- `isRejected = isTagRejected(imageId, tag.label)`

Use a local `useState<boolean>(true)` called `visible` to control collapse animation. When `isAccepted` becomes true, start a `useEffect` timeout of 1500ms to set `visible = false`. When `isRejected` becomes true, set `visible = false` immediately via `useEffect`.

**Card structure (per 006-UI-SPEC lines 302-309):**
```
┌───────────────────────────────────────────────┐
│  tag.label     ████████░░   {n}%  Accept  拒绝  │
└───────────────────────────────────────────────┘
```

Layout: `flex items-center justify-between w-full px-3 py-2` (8px vertical, 12px horizontal per spec line 316).

**Left section — tag label:**
- `text-[12px] font-sans text-text-secondary` (DM Sans 12px per spec line 77)
- No bold, no uppercase

**Middle section — confidence bar + percentage:**
- Mini progress bar: `h-1` (4px per spec line 321), `rounded-[2px]`, full remaining width
- Track: `bg-bg w-full h-full rounded-[2px]`
- Fill: dynamic width `style={{ width: `${tag.confidence}%` }}` rounded-[2px]
- Fill color determined by confidence range (matching 006-UI-SPEC lines 330-334):
  - >80%: `bg-accent` (confidence high — #7a5c12)
  - 50-80%: `bg-text-secondary/30` (moderate)
  - <50%: `bg-text-muted/20` (low)
- Percentage text next to bar: `text-[11px] font-mono` (JetBrains Mono 11px per spec line 77)
- Percentage text color follows the same range: accent / text-secondary / text-muted

**Right section — Accept / Reject buttons:**
- Text-only buttons, no background, no border per spec line 337
- Font: `text-[12px] font-sans`
- Accept idle: `text-success`, hover: `underline text-success`
- Reject idle: `text-text-muted`, hover: `text-danger`
- Gap: 8px between buttons per spec line 341
- On click:
  - Accept: call `store.acceptTag(imageId, tag.label)`
  - Reject: call `store.rejectTag(imageId, tag.label)`

**Card background and border (per 006-UI-SPEC lines 311-318):**
- Idle: `bg-surface border border-border rounded-[2px]`
- Hover: `hover:bg-accent-subtle hover:border-accent/20` with transition `transition-all duration-200 ease-out`
- Accepting state (isAccepted true, visible true): `bg-[rgba(74,122,58,0.05)] border-success/20` (success at 5% opacity per spec line 355)
- Gap between cards: `mt-1` (4px xs spacing per spec line 317 — handled by parent in AiAnalysisSection, not on this component)

**Collapse animation:**
When `visible` becomes false, apply `opacity-0 max-h-0 overflow-hidden transition-all duration-200 ease-out` to animate collapse. Use `pointer-events-none` to prevent interaction during collapse.

**All buttons:** Text-only, accessible name via visible text content. No aria-label needed since text content is always present.

**Keyboard accessibility (per spec lines 372-374):**
- Card itself is `tabIndex={0}` for keyboard focus
- `onKeyDown`: Enter or Space on the card toggles focus, then arrow keys navigate between Accept/Reject

**Export:** `export function TagSuggestionCard({ imageId, tag }: TagSuggestionCardProps)`

**No sub-components.** This is a single self-contained component file.
