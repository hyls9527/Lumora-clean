### Search Input (lines 227-243 of UI-SPEC)

Renders an `<input>` element (NOT the shadcn Input component — the UI-SPEC specifies exact styling that differs from the generic `Input`).

Styling:
- Height: `h-9` (36px)
- Background: `bg-surface`
- Border: `border border-border`
- Focus: `focus:border-accent focus:ring-1 focus:ring-accent/20 focus:outline-none`
- Text: `font-serif text-[14px] text-text` (matches Body role per UI-SPEC line 74 — font-serif for "existing CommandPalette pattern")
- Placeholder: `placeholder:text-text-muted` with value from `t("semanticSearch.placeholder")`
- Border radius: `rounded-[4px]`
- Left padding: `pl-3` (12px)
- Width: `flex-1` to fill available space
- Transitions: `transition-all duration-200 ease-out` on border-color and box-shadow

Behavior:
- `value={query}` and `onChange` handler that calls `setQuery(e.target.value)` from store
- Debounce: Use `useEffect` watching `query`. When `query.length >= 2` and `searchMode === 'semantic'`, start a 300ms `setTimeout` that calls `search(query)`. Cleanup timeout on re-render. When `query.length < 2`, clear results immediately via `clearSearch()`.
- Suggestions: In the same `useEffect`, when `query.length >= 2`, call `loadSuggestions(query)` (no debounce needed — suggestions API is fast per UI-SPEC mock).
- Clear button: Render a `<button>` with text `t("semanticSearch.clear")` when `query.length > 0`. Positioned on the right side of the input (use `absolute right-2` inside the input's relative container). Styling: `font-sans text-[12px] text-text-muted hover:text-text-secondary transition-colors duration-200`. On click: calls `clearSearch()` from store + focuses the input.
- Keyboard: `onKeyDown` handler — Escape clears query (`clearSearch()`), Enter triggers immediate search (`search(query)`).
- Focus management: Use `useRef<HTMLInputElement>` for the input. Expose focus via `window` event listener for ⌘⇧K shortcut (handled in Plan 005-03).

**Searching indicator:** When `isSearching` is true, show subtle text "Searching..." / "搜索中..." (`t("semanticSearch.searching")`) inside the input area. Use `absolute right-10` (to leave room for clear button). Styling: `font-sans text-[12px] text-text-muted`. 200ms fade-in transition.

### Keyboard Handling in the Input

```typescript
const handleKeyDown = (e: React.KeyboardEvent<HTMLInputElement>) => {
  if (e.key === 'Escape') {
    clearSearch()
    inputRef.current?.blur()
  } else if (e.key === 'Enter') {
    search(query)
    // Close dropdown implicitly (blur causes dismiss)
  }
}
```

### Focus and Blur for Dropdown

The dropdown should dismiss when the input loses focus. However, add a 150ms delay before hiding (via `onBlur` + `setTimeout`) to allow click events on suggestion items to register. Track dropdown visibility with local state `showDropdown`.

```typescript
const [showDropdown, setShowDropdown] = useState(false)
const dismissTimeoutRef = useRef<ReturnType<typeof setTimeout> | null>(null)

const handleFocus = () => setShowDropdown(true)
const handleBlur = () => {
  dismissTimeoutRef.current = setTimeout(() => setShowDropdown(false), 150)
}
// Clear timeout on unmount via useEffect cleanup
```

The dropdown renders only when `showDropdown && query.length >= 2 && searchMode === 'semantic' && (suggestions.length > 0 || isLoadingSuggestions)`.
