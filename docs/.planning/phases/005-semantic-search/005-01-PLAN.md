<task type="auto">
  <name>Task 2: Create Semantic Search Zustand Store</name>
  <files>src/stores/semantic-search-store.ts</files>
  <action>
Create `src/stores/semantic-search-store.ts` following the exact Zustand `create<State>` pattern from `src/stores/embedding-store.ts` — interface first, then `create<State>((set, get) => ({...}))`.

Define the `SemanticSearchState` interface:
```typescript
interface SemanticSearchState {
  query: string
  searchMode: 'exact' | 'semantic'
  results: Array<{ imageId: string; score: number }>
  suggestions: string[]
  tryDescribing: string[]
  isSearching: boolean
  isLoadingSuggestions: boolean
  error: string | null

  // Getters
  getScore: (imageId: string) => number | null
  hasResults: () => boolean

  // Actions
  setQuery: (q: string) => void
  setSearchMode: (mode: 'exact' | 'semantic') => void
  search: (query: string) => Promise<void>
  loadSuggestions: (query: string) => Promise<void>
  clearSearch: () => void
}
```

Initial state: `query: ""`, `searchMode: "exact"`, `results: []`, `suggestions: []`, `tryDescribing: []`, `isSearching: false`, `isLoadingSuggestions: false`, `error: null`.

Getters:
- `getScore(imageId)`: Find the result in `results` array by matching `imageId`. Return `result.score` or `null` if not found.
- `hasResults()`: Return `results.length > 0`.

Actions:
- `setQuery(q)`: Call `set({ query: q })`. This is synchronous — debounce is handled by the consumer (SemanticSearchBar), not the store.
- `setSearchMode(mode)`: Call `set({ searchMode: mode })`. When switching to 'exact', also clear results and error: `set({ searchMode: mode, results: [], error: null })`.
- `search(query)`: Set `isSearching: true, error: null`. Then call `await searchSemantic(query)` from the API stub. On success, set `results: data, isSearching: false`. On catch, set `error: String(err), isSearching: false, results: []`. If query is empty string, set `results: [], isSearching: false` immediately without calling API.
- `loadSuggestions(query)`: Set `isLoadingSuggestions: true`. Call `await getSearchSuggestions(query)`. On success, set `suggestions: data.suggestions, tryDescribing: data.tryDescribing, isLoadingSuggestions: false`. On catch, set `isLoadingSuggestions: false` (graceful degradation — suggestions unavailable but search still works). If query < 2 chars, clear suggestions immediately without API call: `set({ suggestions: [], tryDescribing: [], isLoadingSuggestions: false })`.
- `clearSearch()`: Reset to `query: "", results: [], suggestions: [], tryDescribing: [], error: null, isSearching: false`.

Export as: `export const useSemanticSearchStore = create<SemanticSearchState>((set, get) => ({...}))`

All actions that modify state use the functional `set()` pattern with object argument. No `get()` needed for getters since they read from the closure's state. Use `console.error()` in catch blocks matching existing store pattern. No `setTimeout` or `setInterval` in the store — debouncing belongs in the UI layer.
  </action>
