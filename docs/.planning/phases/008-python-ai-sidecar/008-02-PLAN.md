**protocol.rs** — JSON-RPC 2.0 message types using serde:

```rust
use serde::{Deserialize, Serialize};

/// A JSON-RPC 2.0 request sent from Rust to Python.
#[derive(Debug, Clone, Serialize)]
pub struct JsonRpcRequest {
    pub jsonrpc: String,        // always "2.0"
    pub method: String,         // e.g., "health_check", "embed_image", "embed_batch"
    pub params: serde_json::Value, // flexible params — validated by Python handler
    pub id: serde_json::Value,  // string or number — echoed back in response
}

impl JsonRpcRequest {
    pub fn new(method: &str, params: serde_json::Value, id: serde_json::Value) -> Self {
        Self {
            jsonrpc: "2.0".to_string(),
            method: method.to_string(),
            params,
            id,
        }
    }
}

/// A JSON-RPC 2.0 response received from Python.
#[derive(Debug, Clone, Deserialize)]
pub struct JsonRpcResponse {
    pub jsonrpc: String,
    #[serde(default)]
    pub result: Option<serde_json::Value>,
    #[serde(default)]
    pub error: Option<JsonRpcError>,
    pub id: serde_json::Value,
}

#[derive(Debug, Clone, Deserialize)]
pub struct JsonRpcError {
    pub code: i32,
    pub message: String,
}

/// Health check response from Python sidecar (PY-03).
#[derive(Debug, Clone, Deserialize)]
pub struct HealthStatus {
    pub status: String,          // "ok"
    pub model_loaded: bool,
    pub uptime_seconds: f64,
}

/// Single-image embedding result from Python sidecar (PY-01).
#[derive(Debug, Clone, Deserialize)]
pub struct EmbeddingResult {
    pub embedding: Vec<f32>,     // 512 floats
    pub dimensions: u32,         // 512
    pub model: String,           // "ViT-B-32"
}
```

Use `serde_json::Value` for `params` and `id` fields — this gives maximum flexibility. The Python sidecar validates params structure; Rust just passes them through. Use `#[serde(default)]` on optional `result` and `error` fields so that a response with only one of the two parses correctly.

Follow existing codebase patterns: use `serde::{Deserialize, Serialize}` (already in Cargo.toml), Rust 2021 edition conventions. Add doc comments for public types.

**manager.rs** — Sidecar process lifecycle manager:

```rust
use std::sync::Arc;
use tokio::sync::Mutex as TokioMutex;
use tauri::{AppHandle, Manager};
use tauri_plugin_shell::ShellExt;
use tauri_plugin_shell::process::{CommandChild, CommandEvent};
use log::{info, warn, error};

use super::protocol::{JsonRpcRequest, JsonRpcResponse};

/// Manages the Python AI sidecar child process lifecycle.
pub struct SidecarManager {
    child: Arc<TokioMutex<Option<CommandChild>>>,
    healthy: Arc<TokioMutex<bool>>,
}

impl SidecarManager {
    /// Spawn the Python sidecar and begin reading its stdout.
    /// Returns immediately after spawning; model loading happens asynchronously.
    pub fn spawn(app: &AppHandle) -> Result<Self, String> {
        let shell = app.shell();

        // Spawn sidecar using tauri-plugin-shell's sidecar API
        let (mut rx, child) = shell
            .sidecar("binaries/lumora-sidecar")
            .map_err(|e| format!("Failed to create sidecar command: {}", e))?
            .spawn()
            .map_err(|e| format!("Failed to spawn Python sidecar: {}", e))?;

        let child = Arc::new(TokioMutex::new(Some(child)));
        let healthy = Arc::new(TokioMutex::new(false));
        let healthy_clone = healthy.clone();
        let child_clone = child.clone();
        let app_handle = app.clone();

        // Background task: read sidecar stdout, watch for "ready" signal, handle termination
        tauri::async_runtime::spawn(async move {
            while let Some(event) = rx.recv().await {
                match event {
                    CommandEvent::Stdout(line_bytes) => {
                        let line = String::from_utf8_lossy(&line_bytes);
                        // TODO Phase 009: parse JSON-RPC responses and route to awaiting callers
                        info!("Sidecar stdout: {}", line.trim());
                    }
                    CommandEvent::Stderr(line_bytes) => {
                        let line = String::from_utf8_lossy(&line_bytes);
                        // Detect the "[sidecar] ready" signal
                        if line.contains("[sidecar] ready") {
                            info!("Sidecar reported ready — model loaded");
                            *healthy_clone.lock().await = true;
                        }
                        // Log all stderr for debugging
                        info!("Sidecar stderr: {}", line.trim());
                    }
                    CommandEvent::Terminated(status) => {
                        *healthy_clone.lock().await = false;
                        if status.code == Some(0) {
                            info!("Sidecar terminated normally");
                        } else {
                            error!("Sidecar terminated unexpectedly: {:?}", status);
                        }
                        break;
                    }
                    _ => {}
                }
            }
        });

        Ok(Self { child, healthy })
    }

    /// Check if the sidecar responded to the last health ping.
    pub async fn is_healthy(&self) -> bool {
        *self.healthy.lock().await
    }

    /// Send a JSON-RPC request to the sidecar via stdin.
    /// Returns the raw response line from stdout (caller parses it).
    pub async fn send_request(&self, request: &JsonRpcRequest) -> Result<(), String> {
        let mut child_guard = self.child.lock().await;
        let child = child_guard.as_mut().ok_or("Sidecar not running")?;
        
        let json_line = serde_json::to_string(request)
            .map_err(|e| format!("Failed to serialize request: {}", e))?;
        
        child.write((json_line + "\n").as_bytes())
            .map_err(|e| format!("Failed to write to sidecar stdin: {}", e))?;
        
        Ok(())
    }

    /// Gracefully shut down the sidecar process.
    pub async fn shutdown(&self) -> Result<(), String> {
        let mut child_guard = self.child.lock().await;
        if let Some(child) = child_guard.take() {
            child.kill().map_err(|e| format!("Failed to kill sidecar: {}", e))?;
            info!("Sidecar process terminated");
        }
        Ok(())
    }
}
```

Key design decisions (per RESEARCH.md):
- Uses `tauri_plugin_shell::ShellExt` — the Tauri v2 way to spawn sidecars (NOT `std::process::Command`)
- Spawns in background task — does NOT block the Tauri main thread
- Detects "ready" signal from stderr (matching the `[sidecar] ready` line in Plan 01's main.py)
- `send_request()` writes JSON-RPC line to stdin with newline terminator
- `shutdown()` calls `child.kill()` — sidecar has no persistent state to flush
- Uses `Arc<TokioMutex<...>>` for thread-safe access to child handle and health flag
- The stdout reader (TODO Phase 009) will parse JSON-RPC responses and route them to the original caller — for this phase, we log and track health

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Rust→sidecar stdin | JSON-RPC requests cross process boundary — serialization errors must not panic |
| sidecar stdout→Rust | Untrusted input from child process — validate JSON before deserializing |
| Rust state→background task | SidecarManager shared via Arc<Mutex> across async tasks — must not deadlock |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-008-06 | Denial of Service | manager.rs stdout reader | mitigate | CommandEvent stream is bounded; if sidecar floods stdout, the async task drops old events (no unbounded buffering) |
| T-008-07 | Tampering | manager.rs send_request | mitigate | JSON serialization uses serde_json::to_string — malformed requests fail at serialization time, not at the child process |
| T-008-08 | Information Disclosure | manager.rs stderr logging | mitigate | Sidecar stderr is logged at info! level only — no secrets expected in stderr; if model download URL leaks, it's a public HF Hub URL |
| T-008-09 | Elevation of Privilege | tauri.conf.json externalBin | accept | Tauri v2 capability system restricts which binaries can be spawned; externalBin is an allowlist; only lumora-sidecar is whitelisted |
</threat_model>
