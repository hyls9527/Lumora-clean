Create `src-tauri/sidecar/jsonrpc.py` with a `JSONRPCServer` class implementing the JSON-RPC 2.0 line-delimited protocol exactly as specified in RESEARCH.md Pattern 1.

Class API:
- `__init__(self)` — initialize empty `self.methods: dict[str, Callable]`
- `register(self, name: str, handler: Callable[[dict], Any])` — store handler by method name. The handler receives the `params` dict and returns a JSON-serializable result.
- `run(self)` — main loop: iterate `sys.stdin` lines, strip whitespace, skip empty lines, parse JSON, dispatch via `_dispatch`, write response as single-line JSON to `sys.stdout` + flush.
- `_dispatch(self, request: dict) -> dict` — extract method name, lookup handler, call it, format result/error response per JSON-RPC 2.0 spec.

Error codes (use exact values):
- `-32700` Parse error (invalid JSON)
- `-32601` Method not found (unregistered method)
- `-32000` Handler error (exception during handler execution)

Every response includes `"jsonrpc": "2.0"` and mirrors the request's `id`. Notifications (no `id` field) produce NO response — check `if "id" not in request: return`.

Catch `json.JSONDecodeError` for parse errors. Catch generic `Exception` for handler errors — include `str(e)` in the error message. Use `json.dumps(response) + "\n"` for output with immediate `sys.stdout.flush()`.

Do NOT: use async/await (blocking I/O is fine for a single-client sidecar), use logging module (print warnings to stderr directly), implement batch JSON-RPC (array of requests — not needed for our use case).

**model.py** — CLIP model loading + inference:
```python
_model = None
_preprocess = None

def load_model(model_name='ViT-B-32', pretrained='laion2b_s34b_b79k'):
    """Load CLIP model ONCE. Cached on subsequent calls.
    Returns (model, preprocess) tuple. Model is set to eval() mode.
    """
    global _model, _preprocess
    if _model is not None:
        return _model, _preprocess
    import open_clip
    _model, _, _preprocess = open_clip.create_model_and_transforms(model_name, pretrained=pretrained)
    _model.eval()
    return _model, _preprocess

def generate_embedding(image_path):
    """Generate 512-dim L2-normalized CLIP embedding for an image.
    Args: image_path: str — absolute path to image file
    Returns: list[float] — 512 embedding values
    Raises: FileNotFoundError if image_path doesn't exist, ValueError if image can't be opened
    """
    import os, torch
    from PIL import Image
    model, preprocess = load_model()
    if not os.path.isfile(image_path):
        raise FileNotFoundError(f"Image not found: {image_path}")
    # Check file size before opening — reject images > 50MB to prevent OOM (per RESEARCH.md threat model)
    if os.path.getsize(image_path) > 50 * 1024 * 1024:
        raise ValueError(f"Image too large (>50MB): {image_path}")
    image = Image.open(image_path).convert('RGB')
    image_tensor = preprocess(image).unsqueeze(0)  # [1, 3, 224, 224]
    with torch.no_grad():
        embedding = model.encode_image(image_tensor)  # [1, 512]
        embedding = embedding / embedding.norm(dim=-1, keepdim=True)  # L2 normalize
    return embedding.squeeze(0).tolist()
```

Key design decisions (per RESEARCH.md anti-patterns):
- Model loaded ONCE at module level, reused across requests — never reload per request
- CPU-only inference via `torch.no_grad()` — model stays on CPU
- L2 normalization applied after encoding — essential for cosine similarity in vector search
- Image file size guard (50MB max) per threat model
- Uses open_clip's built-in preprocess transform — do NOT hand-roll resize/normalize (Pitfall: preprocessing mismatch)

Do NOT: add GPU support, add batch inference in this task (batch goes in Plan 04), use async inference, bundle the model checkpoint.

**health.py** — health check handler factory:
```python
import time

def create_health_handler(model_loaded_fn):
    """Create a health_check JSON-RPC handler.
    Args: model_loaded_fn: callable that returns bool — whether the CLIP model is loaded
    Returns: handler function suitable for JSONRPCServer.register('health_check', ...)
    """
    start_time = time.time()
    def health_check(params):
        return {
            "status": "ok",
            "model_loaded": model_loaded_fn(),
            "uptime_seconds": round(time.time() - start_time, 1)
        }
    return health_check
```
- `model_loaded_fn` is a lambda/callable that checks if the global model variable in model.py is not None.
- `uptime_seconds` counts from when `create_health_handler` was called (sidecar start time).
- Per RESEARCH.md health check pattern: returns `status`, `model_loaded`, `uptime_seconds`.

**main.py** — entry point:
```python
#!/usr/bin/env python3
"""Lumora Python AI Sidecar — CLIP embedding generation via JSON-RPC 2.0 over stdin/stdout."""
import sys, os

# Ensure sidecar/ directory is on path for imports
sys.path.insert(0, os.path.dirname(os.path.abspath(__file__)))

from jsonrpc import JSONRPCServer
from model import load_model, generate_embedding
from download_model import ensure_model_available
from health import create_health_handler

def main():
    print("[sidecar] Lumora AI Sidecar starting...", file=sys.stderr)
    
    # Step 1: Ensure model checkpoint is available (download if first launch)
    print("[sidecar] Checking CLIP model availability...", file=sys.stderr)
    ensure_model_available(timeout_seconds=600)
    
    # Step 2: Load model into memory
    print("[sidecar] Loading CLIP ViT-B-32 model...", file=sys.stderr)
    model, preprocess = load_model()
    
    # Step 3: Create model-loaded checker for health endpoint
    def is_model_loaded():
        return model is not None
    
    # Step 4: Create JSON-RPC server and register handlers
    server = JSONRPCServer()
    server.register("health_check", create_health_handler(is_model_loaded))
    server.register("embed_image", lambda params: {
        "embedding": generate_embedding(params["image_path"]),
        "dimensions": 512,
        "model": "ViT-B-32"
    })
    
    # Step 5: Ready — signal to Tauri parent process
    print("[sidecar] ready", file=sys.stderr)
    sys.stderr.flush()
    
    # Step 6: Run the JSON-RPC stdin/stdout event loop (blocks forever)
    server.run()

if __name__ == "__main__":
    main()
```

Key design:
- The `[sidecar]` prefix on all stderr messages lets the Rust manager distinguish sidecar logs from other stderr output.
- The "ready" signal on stderr is the contract with the Rust manager (Plan 02) — Rust waits for this line before sending requests.
- `embed_image` handler wraps `generate_embedding()` — returns both the embedding vector and metadata (dimensions, model name) per the JSON-RPC response format in RESEARCH.md.
- Model download timeout is 600 seconds (10 minutes) — for 605MB over slow connections. See RESEARCH.md Pitfall 4.
- `sys.path.insert(0, ...)` at the top of main.py ensures imports work both during development (`py main.py`) and inside the PyInstaller bundle.

Do NOT: add batch methods here (Plan 04 adds those), add signal handlers for graceful shutdown (Phase 012 packaging handles this), use argparse (no CLI arguments — everything is JSON-RPC).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tauri→stdin | JSON-RPC requests cross from Tauri parent process into Python sidecar — Rust controls the input, but image_path values come from user file system |
| Python→stdout | JSON-RPC responses cross from Python back to Rust — Rust must validate before deserializing |
| Python→filesystem | Sidecar reads image files from user's filesystem — arbitrary paths could be passed |
| Python→network | Model download accesses Hugging Face Hub on first launch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-008-01 | Tampering | model.py:generate_embedding image_path | mitigate | Validate path exists with os.path.isfile(); reject paths with '..' segments after realpath resolution; reject image files >50MB |
| T-008-02 | Denial of Service | jsonrpc.py:_dispatch | mitigate | Catch all exceptions in handler dispatch; return JSON-RPC error, never crash; validate JSON before parsing large payloads |
| T-008-03 | Denial of Service | model.py:generate_embedding | mitigate | Check file size before opening (50MB limit); Pillow's Image.open() does not load full image into memory by default — only pixel data on access |
| T-008-04 | Tampering | download_model.py checkpoint | accept | Model checkpoint downloaded from official Hugging Face Hub (mlfoundations/open_clip). Risk accepted for v0.3; SHA256 verification deferred to Phase 012. |
| T-008-05 | Elevation of Privilege | download_model.py pickle checkpoint | accept | open-clip uses safetensors format by default (not pickle); pickle-based checkpoints are legacy. Risk accepted for now; verify safetensors usage during build. |
</threat_model>
