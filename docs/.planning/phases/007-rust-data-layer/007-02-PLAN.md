**Step B: Create `src/lib/tauri.ts`**
Create the file with a robust `isTauri()` detection function:

```typescript
let _isTauri: boolean | null = null

export function isTauri(): boolean {
  if (_isTauri !== null) return _isTauri
  try {
    // Dynamic import to avoid bundling Tauri API in non-Tauri builds
    _isTauri =
      typeof window !== 'undefined' &&
      '__TAURI_INTERNALS__' in window
  } catch {
    _isTauri = false
  }
  return _isTauri
}

export function resetIsTauri(): void {
  _isTauri = null
}
```

This pattern:
- Uses `window.__TAURI_INTERNALS__` for detection (the canonical Tauri v2 runtime signal). Does NOT import `@tauri-apps/api` at module level — this avoids errors when running in a pure browser.
- Caches the result so repeated calls are free.
- Has a `resetIsTauri()` for testing or if the runtime changes.
- Returns `false` in browser mode (no `__TAURI_INTERNALS__`), `true` when running inside a Tauri window.

Do NOT use `@tauri-apps/api/core`'s `isTauri()` import directly — that import fails in non-Tauri contexts. The `window.__TAURI_INTERNALS__` check is the standard approach.

Note: `@tauri-apps/api/core` will be imported LAZILY in Phase 9 when actual Tauri IPC functions are needed. The `isTauri()` guard ensures those imports only execute inside Tauri.

| T-007-02b | Information Disclosure | isTauri() caching | accept | The boolean `isTauri` result contains no sensitive data. The cached value is a simple runtime check. |
