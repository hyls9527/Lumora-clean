**Step B: Create capabilities/default.json**
Create `src-tauri/capabilities/` directory and `default.json`:

```json
{
  "identifier": "default",
  "description": "Default capability for the main window",
  "windows": ["main"],
  "permissions": [
    "core:default",
    "store:default",
    "dialog:default",
    "core:window:default",
    "core:event:default"
  ]
}
```

This whitelists:
- `core:default` — Basic IPC
- `store:default` — tauri-plugin-store read/write
- `dialog:default` — Native file/folder picker dialogs
- `core:window:default` — Window management
- `core:event:default` — Event system (for import progress events)

Note: Tauri v2 capabilities use a whitelist model. Commands registered in `generate_handler![]` are available to windows that have the `core:default` permission. Custom command-level permissions are not needed for Phase 7's commands.

Per D-22: No additional config needed for portable data directory — that's handled in Rust code (Plan 07 `filesystem/paths.rs`).
