## 缩略图生成

### 生成时机
| Option | Description | Selected |
|--------|-------------|----------|
| 导入时立刻生成 | 画廊打开就是完整的，不卡顿浏览 | ✓ |
| 后台按需生成 | 导入瞬间完成，但画廊会有加载空白 | |

**Notes:** 用户偏好即时完成，不接受空白卡片。

### 质量
| Option | Description | Selected |
|--------|-------------|----------|
| 高质量 ~80KB | 画廊清晰锐利，但占磁盘 | |
| 均衡 ~40KB | 够清晰，磁盘合理 | |
| 轻量 ~20KB | 磁盘最小，详情面板看原图 | ✓ |

**User's choice:** "点击详情就是原图画质，对吧？" — 确认详情面板显示原图，因此缩略图可用轻量级。

### 生成失败处理
| Option | Description | Selected |
|--------|-------------|----------|
| 跳过并标记 | 失败图片打标记，其他正常 | |
| 跳过不导入 | 坏图片不进入画廊 | |
| 导入但不显示 | 后台存在但画廊隐藏 | |

**User's choice:** "应该有重新导入" — 标记失败 + 提供重新生成按钮。

### 存放方式
| Option | Description | Selected |
|--------|-------------|----------|
| 存为单独文件 | 数据库轻量，文件直观 | ✓ |
| 存进数据库 | 打包在一起 | |

**User's choice:** 存为单独文件。用户指出"备份时，你只备份缩略图吗？"——原图本身就是文件，缩略图存数据库无优势。

### 批量导入进度
| Option | Description | Selected |
|--------|-------------|----------|
| 显示整体进度 | "正在处理 12/50" 进度条 | ✓ |
| 逐张显示 | 每张依次出现 | |

### 特殊格式（BMP/TIFF）
| Option | Description | Selected |
|--------|-------------|----------|
| 导入时转成 PNG | 浏览器兼容 | ✓ |
| 保留原格式 | 不丢原始数据 | |

### 支持格式
| Option | Description | Selected |
|--------|-------------|----------|
| 主流三件 JPG+PNG+WebP | 覆盖 95%，实现简单 | |
| 全格式支持 | 用户拖什么都行 | ✓ |
