<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Install Rust Toolchain and MSVC Build Tools</name>
  <what-built>System-level Rust development environment on Windows 11</what-built>
  <how-to-verify>
**Step 1: Install Rust via rustup**
1. Open PowerShell or Command Prompt as administrator
2. Visit https://rustup.rs and download `rustup-init.exe`
3. Run the installer. When prompted, select option `1` (default installation)
4. IMPORTANT: The default toolchain MUST be `stable-msvc` (not `stable-gnu`). If asked, choose the MSVC host triple.
5. After installation, RESTART your terminal (or run `refreshenv` if using Chocolatey)

**Step 2: Verify Rust installation**
Run these commands in a NEW terminal:
```
rustc --version
cargo --version
rustup show
```
Expected: `rustc 1.85.0` or later, `cargo 1.85.0` or later, default toolchain shows `stable-x86_64-pc-windows-msvc`.

**Step 3: Test basic compilation**
Create and build a test project:
```
cd %TEMP%
cargo new test_rust
cd test_rust
cargo build
```
Expected: Compilation succeeds with zero errors. If it fails with linker errors (e.g., `LINK : fatal error LNK1181`), MSVC Build Tools are missing — proceed to Step 4.

**Step 4: Install MSVC Build Tools (if linker errors in Step 3)**
1. Visit https://visualstudio.microsoft.com/downloads/#build-tools-for-visual-studio-2022
2. Download "Build Tools for Visual Studio 2022"
3. Run the installer. Under "Workloads", select "Desktop development with C++"
4. In the "Individual components" tab, ensure these are checked:
   - MSVC v143 - VS 2022 C++ x64/x86 build tools
   - Windows 11 SDK (or Windows 10 SDK)
   - C++ CMake tools for Windows
5. Click Install. Wait for completion (5-15 minutes).
6. Re-run the test compilation from Step 3.
  </how-to-verify>
  <resume-signal>After ALL steps above succeed (rustc, cargo, and test compilation all pass), type "approved" to continue. If any step fails, describe the error message.</resume-signal>
</task>

<task type="auto">
  <name>Task 2: Install Tauri CLI v2</name>
  <files>None (system-level cargo install)</files>
  <action>
Install the Tauri CLI v2 via cargo. This is a one-time system-level install.

Run: `cargo install tauri-cli --version "^2"`

This compiles and installs the `cargo tauri` subcommand. Compilation may take 5-10 minutes as it builds all Tauri CLI dependencies from source.

After installation, verify:
```
cargo tauri --version
```
Expected: Output shows `tauri-cli` version 2.x (e.g., `tauri-cli 2.4.x`).

If the install fails with compilation errors, try: `cargo install tauri-cli --version "^2" --locked`

The `--locked` flag uses the exact dependency versions from the Cargo.lock shipped with the crate, avoiding potential incompatibilities with newer transitive dependencies.

Do NOT install `@tauri-apps/cli` via npm — the Rust cargo-installed CLI is required for `cargo tauri init`.
  </action>
  <verify>
    <automated>cargo tauri --version</automated>
  </verify>
  <done>`cargo tauri --version` outputs version 2.x. `cargo tauri --help` shows available subcommands (init, dev, build, etc.).</done>
</task>
