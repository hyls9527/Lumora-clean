fn get_image_tags(conn: &rusqlite::Connection, image_id: &str) -> Result<Vec<String>, rusqlite::Error> {
    let mut stmt = conn.prepare(
        "SELECT t.name FROM tags t
         JOIN image_tags it ON t.name = it.tag_name
         WHERE it.image_id = ?1
         ORDER BY t.name"
    )?;
    let tags: Vec<String> = stmt.query_map(params![image_id], |row| row.get(0))?
        .filter_map(|r| r.ok())
        .collect();
    Ok(tags)
}
```

pub fn list_images(
    state: State<'_, Mutex<AppState>>,
    limit: u32,
    offset: u32,
) -> Result<Vec<ImageRecord>, AppError> {
    let app_state = state.lock().map_err(|e| AppError::Database(e.to_string()))?;

    let rows = db::images::list_images(conn, limit, offset)?;

        let mut record = ImageRecord::from(row);

        records.push(record);
    }

    Ok(records)
}

#[tauri::command]
pub fn update_image(
    state: State<'_, Mutex<AppState>>,
    id: String,
    payload: UpdateImagePayload,
) -> Result<(), AppError> {
    let app_state = state.lock().map_err(|e| AppError::Database(e.to_string()))?;
    let conn = &app_state.conn;

    // Verify image exists
    if db::images::get_image_by_id(conn, &id)?.is_none() {
        return Err(AppError::NotFound(format!("Image not found: {}", id)));
    }

    if let Some(rating) = payload.rating {
        db::images::set_image_rating(conn, &id, rating)?;
    }
    if let Some(favorite) = payload.favorite {
        db::images::set_image_favorite(conn, &id, favorite)?;
    }

    Ok(())
}
```

**`import_images` command (async with spawn_blocking):**

This is the primary async command. Per RESEARCH.md Pattern 3, it uses `tokio::task::spawn_blocking` for all heavy work (image metadata reading, DB inserts). Progress events are emitted via `app.emit()`.

IMPORTANT: This is a BASIC import — it inserts metadata into the database but does NOT copy files, generate thumbnails, or compute hashes. Those file operations are added in Plan 07. The command signature and flow are structured so Plan 07 extends them without structural changes.

```rust
#[tauri::command]
pub async fn import_images(
    app: tauri::AppHandle,
    state: State<'_, Mutex<AppState>>,
    paths: Vec<String>,
) -> Result<ImportResult, AppError> {
    let app_for_events = app.clone();

    tokio::task::spawn_blocking(move || {
        let mut app_state = state.lock().map_err(|e| AppError::Database(e.to_string()))?;
        let conn = &app_state.conn;
        let total = paths.len() as u32;
        let mut result = ImportResult {
            imported: 0,
            skipped_duplicates: 0,
            skipped_non_image: 0,
            skipped_too_large: 0,
            errors: Vec::new(),
        };

        for (i, path_str) in paths.iter().enumerate() {
            let path = std::path::Path::new(path_str);

            // Skip non-image files per D-15
            if !is_supported_image(path) {
                result.skipped_non_image += 1;
                let _ = app_for_events.emit("import-progress", ImportProgress {
                    current: i as u32 + 1,
                    total,
                    file_name: path_str.clone(),
                });
                continue;
            }

            // Extract original file name
            let original_name = path
                .file_name()
                .and_then(|n| n.to_str())
                .unwrap_or("unknown")
                .to_string();

            // Read basic image metadata using the image crate
            match image::image_dimensions(path) {
                Ok((width, height)) => {
                    let file_size = match std::fs::metadata(path) {
                        Ok(m) => m.len() as i64,
                        Err(_) => 0,
                    };

                    let format = path.extension()
                        .and_then(|e| e.to_str())
                        .map(|e| e.to_lowercase())
                        .unwrap_or_else(|| "unknown".to_string());

                    // Generate UUID for the image
                    let id = uuid::Uuid::new_v4().to_string();

                    // Insert image metadata into database (file copy/thumbnail in Plan 07)
                    let new_image = db::models::NewImage {
                        id: id.clone(),
                        file_path: path_str.clone(),
                        original_name,
                        width: width as i32,
                        height: height as i32,
                        file_size,
                        format,
                        thumbnail_path: None, // Plan 07 adds thumbnail path
                        sha256_hash: None,    // Plan 07 adds hash
                    };

                    match db::images::insert_image(conn, &new_image) {
                        Ok(()) => {
                            result.imported += 1;
                        }
                        Err(e) => {
                            result.errors.push(format!("Failed to insert {}: {}", path_str, e));
                        }
                    }
                }
                Err(e) => {
                    // Could not read image dimensions — skip or error
                    result.errors.push(format!("Failed to read {}: {}", path_str, e));
                }
            }

            // Emit progress event per D-05
            let _ = app_for_events.emit("import-progress", ImportProgress {
                current: i as u32 + 1,
                total,
                file_name: path_str.clone(),
            });
        }

        // Emit completion event
        let _ = app_for_events.emit("import-complete", &result);
        Ok(result)
    })
    .await
    .map_err(|e| AppError::Io(e.to_string()))?
}
```

**Key design decisions for Plan 07 extension:**
- The `import_images` command reads `image_dimensions` but does NOT copy files, generate thumbnails, or hash. These are stubbed as `thumbnail_path: None, sha256_hash: None`.
- Plan 07 will add a file processing step AFTER the dimension read and BEFORE the DB insert, reusing the same loop structure.
- The `ImportProgress` event emitted per file keeps the frontend UI informed per D-05 ("12/50").
- Duplicate detection (`sha256_hash` check) is NOT functional until Plan 07 adds hashing.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend invoke() → Rust command | User-supplied parameters (search query, image ID, rating value) cross from untrusted JS to Rust |
| File path → image crate | User-provided file paths are passed to `image::image_dimensions()` and `std::fs::metadata()` — path traversal risk |
| Rust command → SQLite | Rusqlite parameterized queries protect against SQL injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-007-05a | Tampering | FTS5 MATCH query injection | mitigate | Use `params![query]` — rusqlite bind parameters prevent FTS5 query injection. User input is never concatenated into SQL strings. |
| T-007-05b | Tampering | Path traversal in import_images | mitigate | Validate file extension against whitelist. `std::path::Path` methods normalize paths. Phase 9 adds `std::fs::canonicalize()` check before passing paths to Rust. |
| T-007-05c | Tampering | Rating value tampering | mitigate | Validate `rating` in range 0-5. Return `AppError::Validation` for out-of-range values. |
| T-007-05d | Information Disclosure | Error messages contain file paths | accept | Error messages include file paths for debugging. This is a local desktop app — error messages are only visible in logs, not exposed to remote users. |
| T-007-05e | Denial of Service | Large limit parameter in list_images | mitigate | Clamp `limit` to 1-200 range. |
| T-007-SC | Tampering | Package legitimacy | accept | No new packages added. |
