# Plan 005-01: Tauri Initialization

## Task 1: Initialize Tauri in the project

<action>
Run the Tauri init command to set up the Rust backend:
```bash
npm install @tauri-apps/cli @tauri-apps/api
npx tauri init
```

This will create:
- `src-tauri/` directory with Cargo.toml, tauri.conf.json, src/main.rs, src/lib.rs
- Update package.json with Tauri scripts
</action>

<acceptance_criteria>
- src-tauri/ directory exists with proper structure
- Cargo.toml has tauri dependency
- tauri.conf.json has correct app info
- package.json has tauri scripts (dev, build)
</acceptance_criteria>

## Task 2: Configure Tauri for Lumora

<action>
Update tauri.conf.json with Lumora-specific settings:

```json
{
  "productName": "Lumora",
  "version": "0.1.0",
  "identifier": "com.lumora.app",
  "build": {
    "frontendDist": "../dist",
    "devUrl": "http://localhost:5173",
    "beforeDevCommand": "npm run dev",
    "beforeBuildCommand": "npm run build"
  },
  "app": {
    "title": "Lumora — 光之韵律",
    "windows": [
      {
        "title": "Lumora — 光之韵律",
        "width": 1200,
        "height": 800,
        "minWidth": 800,
        "minHeight": 600,
        "center": true,
        "resizable": true,
        "decorations": true
      }
    ],
    "security": {
      "csp": null
    }
  },
  "bundle": {
    "active": true,
    "targets": "all",
    "icon": [
      "icons/32x32.png",
      "icons/128x128.png",
      "icons/128x128@2x.png",
      "icons/icon.icns",
      "icons/icon.ico"
    ]
  }
}
```
</action>

<acceptance_criteria>
- App title is "Lumora — 光之韵律"
- Window size is 1200x800 with min 800x600
- Frontend dist path is correct
- Bundle icons are configured
</acceptance_criteria>

## Task 3: Update package.json scripts

<action>
Add Tauri-specific scripts to package.json:
```json
{
  "scripts": {
    "tauri": "tauri",
    "tauri:dev": "tauri dev",
    "tauri:build": "tauri build"
  }
}
```
</action>

<acceptance_criteria>
- `npm run tauri:dev` starts the dev server
- `npm run tauri:build` builds the app
- Scripts are correctly configured
</acceptance_criteria>
