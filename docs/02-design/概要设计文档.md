# 概要设计文档 — Lumora

> **审计原则**：每个设计决策必须追溯到具体代码。

## 1. 系统架构概述

Lumora 采用前后端分离的桌面应用架构：

```
┌─────────────────────────────────────────────────────────────┐
│                      Frontend (React)                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐    │
│  │  Gallery  │  │  Search  │  │ Dashboard │  │ Settings │    │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘    │
│         │             │             │             │          │
│  ┌──────────────────────────────────────────────────────┐   │
│  │              Zustand Stores                           │   │
│  │  imageStore | trashStore | settingsStore | ...        │   │
│  └──────────────────────────────────────────────────────┘   │
│         │                                                    │
│  ┌──────────────────────────────────────────────────────┐   │
│  │              API Layer (lib/api/)                      │   │
│  │  images.ts | ai.ts | embeddings.ts | semantic.ts      │   │
│  └──────────────────────────────────────────────────────┘   │
│         │                                                    │
│  ┌──────────────────────────────────────────────────────┐   │
│  │              Cache Layer (lib/api/semanticCache.ts)    │   │
│  │  Map-based LRU | 30min TTL | 5MB eviction | debounce  │   │
│  └──────────────────────────────────────────────────────┘   │
│         │                                                    │
│  ┌──────────────────────────────────────────────────────┐   │
│  │              Hooks                                    │   │
│  │  useOllamaStatus (60s poll) | useKeyboardNav | ...    │   │
│  └──────────────────────────────────────────────────────┘   │
│         │                                                    │
│  ┌──────────────────────────────────────────────────────┐   │
│  │              Tauri Invoke Wrapper (lib/tauri.ts)       │   │
│  │  Browser mode: mock data | Tauri mode: real invoke    │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────┬───────────────────────────────────┘
                          │ IPC (invoke)
┌─────────────────────────┴───────────────────────────────────┐
│                    Backend (Rust/Tauri)                       │
│  ┌──────────────────────────────────────────────────────┐   │
│  │              Tauri Commands                            │   │
│  │  images | ai | embeddings | tags | trash | dashboard  │   │
│  │  export | settings | backup | clip | ollama          │   │
│  └──────────────────────────────────────────────────────┘   │
│         │                                                    │
│  ┌──────────────────────────────────────────────────────┐   │
│  │              Database Layer (db/)                      │   │
│  │  SQLite + FTS5 + sqlite-vec | Migrations v1-v6       │   │
│  └──────────────────────────────────────────────────────┘   │
│         │                                                    │
│  ┌──────────────────────────────────────────────────────┐   │
│  │              External Services                         │   │
│  │  Ollama API (localhost:11434)                          │   │
│  │  - nomic-embed-text (embedding)                        │   │
│  │  - llava (image analysis)                              │   │
│  │  - /api/tags (health check, polled by useOllamaStatus) │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

**代码依据**：
- 前端入口：`src/App.tsx`
- 后端入口：`src-tauri/src/lib.rs`
- IPC 命令注册：`lib.rs` invoke_handler

## 2. 分层架构

### 2.1 前端分层

| 层级 | 职责 | 文件位置 |
|------|------|----------|
| **页面层** | 路由、布局、页面级状态 | `src/features/*/` |
| **组件层** | UI 组件、交互逻辑 | `src/components/ui/` |
| **Store 层** | 全局状态管理 (Zustand) | `src/stores/` |
| **API 层** | IPC 调用、数据转换 | `src/lib/api/` |
| **缓存层** | LRU 缓存、持久化 | `src/lib/api/semanticCache.ts` |
| **工具层** | i18n、tokens、helpers | `src/lib/` |

**代码依据**：
- Store 示例：`imageStore.ts`, `trashStore.ts`, `aiAnalysisStore.ts`
- API 示例：`images.ts`, `ai.ts`, `embeddings.ts`
- 缓存：`semanticCache.ts`

### 2.2 后端分层

| 层级 | 职责 | 文件位置 |
|------|------|----------|
| **命令层** | Tauri IPC 命令处理 | `src-tauri/src/commands/` |
| **业务层** | 业务逻辑 | `src-tauri/src/commands/*/` |
| **数据层** | 数据库操作 | `src-tauri/src/db/` |
| **Schema 层** | 类型定义 | `src-tauri/src/schema/` |
| **工具层** | 元数据解析、Ollama 集成 | `src-tauri/src/metadata/`, `src-tauri/src/ollama.rs` |

**代码依据**：
- 命令模块：`commands/images/`, `commands/tags/`, `commands/trash/` 等
- 数据库：`db/mod.rs`, `db/schema.rs`, `db/migrations.rs`
- 元数据：`metadata/mod.rs`, `metadata/sd.rs`, `metadata/png.rs`

## 3. 数据流设计

### 3.1 读操作流程

```
用户操作 → 组件 → Store.fetchXxx() → API.xxx() → invoke('xxx_cmd') → Rust Command → SQLite → 返回结果
```

**代码依据**：
- `imageStore.fetchImages()` → `api.listImages()` → `invoke('list_images')`

### 3.2 写操作流程

```
用户操作 → 组件 → Store.xxx() → API.xxx() → invoke('xxx_cmd') → Rust Command → SQLite → 返回结果
                                    ↓
                              乐观更新 UI
                                    ↓
                              失败时回滚
```

**代码依据**：
- `trashStore.restoreImage()` 乐观更新 + 失败回滚

### 3.3 缓存策略

| 缓存类型 | 策略 | 代码位置 |
|----------|------|----------|
| 语义搜索缓存 | Map-based LRU, 30 分钟 TTL, 200 条上限 | `semanticCache.ts` |
| localStorage 持久化 | 防抖写入 (300ms) | `semanticCache.ts` |
| 写命令监听 | 写操作后清除缓存 | `onWriteCommand()` |

## 4. 状态管理设计

### 4.1 Store 结构

每个 Store 遵循统一模式：

```typescript
interface XxxStore {
  // State
  data: DataType[];
  loading: boolean;
  error: string | null;
  
  // Async Actions
  fetchData: () => Promise<void>;
  
  // Sync Actions
  setFilter: (filter: FilterType) => void;
}
```

**代码依据**：
- `imageStore.ts`: 完整示例
- `trashStore.ts`: 乐观更新示例
- `aiAnalysisStore.ts`: Record<key, value> 结构示例

### 4.2 Store 列表

| Store | 职责 | 文件 |
|-------|------|------|
| `imageStore` | 图片列表、过滤、搜索 | `src/stores/imageStore.ts` |
| `imageSearchStore` | 以图搜图状态 | `src/stores/imageSearchStore.ts` |
| `imageTagsStore` | 标签 CRUD | `src/stores/imageTagsStore.ts` |
| `trashStore` | 回收站操作 | `src/stores/trashStore.ts` |
| `aiAnalysisStore` | AI 分析结果 | `src/stores/aiAnalysisStore.ts` |
| `embeddingStore` | 嵌入状态 | `src/stores/embeddingStore.ts` |
| `semanticSearchStore` | 语义搜索 | `src/stores/semanticSearchStore.ts` |
| `settingsStore` | 应用设置 | `src/stores/settingsStore.ts` |

## 5. 设计系统

### 5.1 Token 系统

所有颜色使用 CSS 自定义属性，支持暗色主题切换：

```typescript
// src/lib/tokens.ts
export const t = {
  bg: 'var(--color-bg)',
  text: 'var(--color-text)',
  accent: 'var(--color-accent)',
  // ...
};
```

**代码依据**：
- `src/lib/tokens.ts`: 完整 token 定义
- `src/index.css`: CSS 变量定义 + `[data-theme="dark"]` 覆盖

### 5.2 字体系统

| 用途 | 字体 | CSS 变量 |
|------|------|----------|
| 显示字体 | `var(--font-display)` | 标题、标签 |
| 正文字体 | `var(--font-body)` | 正文、描述 |

### 5.3 主题切换

- 默认主题：暗色 ("暗夜")
- 切换方式：`data-theme` 属性
- 实现：CSS 变量覆盖

**代码依据**：
- `src/index.css`: `[data-theme="dark"]` 块

## 8. 性能优化设计

### 8.1 前端优化

| 优化策略 | 实现方式 | 代码位置 |
|----------|----------|----------|
| 懒加载 | `React.lazy()` + `Suspense` | `App.tsx` |
| 虚拟滚动 | `LazyLoad` 组件 | `LazyLoad.tsx` |
| CSS containment | `contain: strict` | CSS |
| LRU 缓存 | Map-based, 30 分钟 TTL | `semanticCache.ts` |
| 防抖写入 | 300ms 延迟持久化 | `semanticCache.ts` |

### 8.2 后端优化

| 优化策略 | 实现方式 | 代码位置 |
|----------|----------|----------|
| FTS5 索引 | 全文搜索 | `db/schema.rs` |
| 向量索引 | sqlite-vec | `db/schema.rs` |
| 分页查询 | LIMIT/OFFSET | `commands/images/` |
