# 详细设计文档 — Lumora v0.5.1

## 1. IPC 命令详细设计

### 1.1 import_images

```rust
// 签名
fn import_images(db: State<DbHandle>, path: String) -> AppResult<ImportResult>

// 返回值
struct ImportResult {
    imported: u32,      // 成功导入数
    skipped: u32,       // 跳过数（重复文件）
    total_scanned: u32, // 扫描总数
    items: Vec<ImageRecord>,
}

// 处理流程
1. scan_folder(path) → Vec<ImportEntry>
   - 单文件模式：直接处理
   - 文件夹模式：walk_dir() 递归扫描
   - 过滤：仅处理已知图片扩展名
2. 对每个 entry：
   - file_hash(path, size) → 去重
   - probe_dimensions_from_bytes() → 提取尺寸
   - metadata::probe_metadata_from_bytes() → 提取元数据
   - find_or_create_variant_group() → 关联变体组
3. 事务内批量 INSERT
4. 返回 ImportResult
```

### 1.2 search_semantic_cmd

```rust
// 签名
fn search_semantic_cmd(
    db: State<DbHandle>,
    query_embedding: Vec<f32>,
    limit: Option<usize>,
) -> AppResult<Vec<SemanticSearchResult>>

// 返回值
struct SemanticSearchResult {
    image_id: String,
    similarity: f32,  // 0.0 ~ 1.0
}

// 处理流程
1. 查询 vec_embeddings 虚拟表
2. 使用 cosine distance 计算相似度
3. 按相似度排序，取 top N
4. 返回结果
```

### 1.3 analyze_image_cmd

```rust
// 签名
async fn analyze_image_cmd(
    db: State<DbHandle>,
    ollama: State<OllamaConfig>,
    image_id: String,
    image_path: String,
    model: Option<String>,
) -> AppResult<AnalysisResult>

// 处理流程
1. 读取图片文件 → base64 编码
2. 构造 Ollama API 请求（model=llava, timeout=120s）
3. 发送 POST /api/generate
4. 解析响应 → AnalysisResult
5. INSERT INTO analysis_history
6. 返回结果
```

## 3. 缓存设计

### semanticCache

```typescript
// Map-based LRU with persistence
const cache = new Map<string, CacheEntry>();

interface CacheEntry {
  results: SemanticSearchResult[];
  timestamp: number;    // 用于 TTL 检查
  accessOrder: number;  // 用于 LRU 排序
}

// 策略
- TTL: 30 分钟
- 最大条目: 100
- 最大大小: 5MB (JSON 序列化后)
- 持久化: localStorage (延迟写入，防抖 2s)
- 失效: 写命令后自动清除（通过 onWriteCommand 回调）
```

## 4. 错误处理设计

### 4.1 后端错误类型

```rust
enum AppError {
    Io(std::io::Error),           // 文件 IO 错误
    Db(rusqlite::Error),          // 数据库错误
    Lock,                         // 互斥锁获取失败
    NotFound(String),             // 资源不存在
    Ollama(String),               // Ollama API 错误
}
```

### 4.2 前端错误处理

```
invoke 调用失败
  → catch 块记录 console.error
  → 映射为 i18n 错误消息
  → store 设置 error 状态
  → UI 展示 ErrorState 组件（含重试按钮）
```
