# 发布流程文档 — Lumora

> **审计原则**：每个发布步骤必须追溯到具体配置和命令。

## 1. 发布类型

| 类型 | 触发条件 | 说明 |
|------|----------|------|
| 正式版 | `v*` tag | 生产发布 |
| 预发布 | `v*-rc*` tag | 候选版本 |
| 热修复 | `v*-hotfix*` tag | 紧急修复 |

**代码依据**：`.github/workflows/release.yml` on.push.tags

## 2. 发布前检查

### 2.1 版本同步

**必须同步的文件**：

| 文件 | 字段 | 位置 |
|------|------|------|
| `package.json` | `version` | 根目录 |
| `Cargo.toml` | `version` | `src-tauri/` |
| `tauri.conf.json` | `version` | `src-tauri/` |

**检查命令**：
```bash
# 手动检查
grep '"version"' package.json
grep '^version' src-tauri/Cargo.toml
grep '"version"' src-tauri/tauri.conf.json

# CI 自动检查（release.yml verify job）
```

---

### 2.2 质量门禁

```bash
# 前端
npx tsc --noEmit          # TypeScript 类型检查
npx vitest run            # 前端测试

# 后端
cd src-tauri
cargo test --lib          # Rust 测试
cargo check               # Rust 编译检查
```

---

### 2.3 更新日志

**文件**：`CHANGELOG.md`

**格式**：

```markdown
### Added

- 修复描述

### Changed
- 变更描述
```

**代码依据**：`CHANGELOG.md`

## 3. 发布步骤

### 3.1 准备阶段

1. **更新版本号**
   ```bash
   # 更新 package.json
   npm version X.Y.Z --no-git-tag-version
   
   # 更新 Cargo.toml
   cd src-tauri
   cargo version X.Y.Z
   
   # 更新 tauri.conf.json
   # 手动编辑 version 字段
   ```

2. **更新 CHANGELOG.md**
   - 添加新版本条目
   - 记录所有变更

3. **提交变更**
   ```bash
   git add .
   git commit -m "chore: prepare release vX.Y.Z"
   ```

---

### 3.2 发布阶段

4. **创建 Tag**
   ```bash
   git tag vX.Y.Z
   git push origin vX.Y.Z
   ```

5. **触发 CI**
   - GitHub Actions 自动触发 `release.yml`
   - 执行以下步骤：
     - 版本同步检查
     - 前端测试
     - Rust 测试
     - 三平台构建
     - 签名
     - 上传产物
     - 生成 updater JSON

---

### 3.3 验证阶段

6. **检查 GitHub Release**
   - 访问 `https://github.com/hyls9527/Lumora-clean/releases`
   - 确认 Draft Release 已创建
   - 确认所有平台产物已上传

7. **下载测试**
   - 下载各平台安装包
   - 安装并测试基本功能
   - 验证自动更新

8. **发布 Release**
   - 编辑 Release Notes
   - 点击 "Publish release"

## 5. 自动更新流程

### 5.1 更新检查

```
客户端启动
    ↓
检查 https://github.com/.../latest.json
    ↓
比较本地版本与远程版本
    ↓
如有更新，显示 UpdateBanner
    ↓
用户点击更新
    ↓
下载签名文件
    ↓
验证签名 (minisign)
    ↓
下载更新包
    ↓
应用更新
```

---

### 5.2 updater JSON

**生成时机**：`release.yml` updater-json job

**上传位置**：GitHub Release assets

**格式**：
```json
{
  "version": "X.Y.Z",
  "notes": "",
  "pub_date": "ISO8601",
  "platforms": {
    "windows-x86_64": { "signature": "...", "url": "..." },
    "darwin-aarch64": { "signature": "...", "url": "..." },
    "linux-x86_64": { "signature": "...", "url": "..." }
  }
}
```

**代码依据**：`.github/workflows/release.yml` updater-json job

## 6. 签名配置

```bash
# 生成 minisign 密钥对
```

## 7. 回滚流程

### 7.1 回滚步骤

1. **删除 GitHub Release**
   - 访问 Release 页面
   - 删除问题版本的 Release

2. **删除 Tag**
   ```bash
   git tag -d vX.Y.Z
   git push origin :refs/tags/vX.Y.Z
   ```

3. **修复问题**
   - 修复代码
   - 提交并推送

4. **重新发布**
   - 按正常流程重新发布

---

### 7.2 紧急回滚

如果用户已更新到问题版本：

1. 发布修复版本 vX.Y.Z+1
2. 在 Release Notes 中说明问题
3. 用户通过自动更新获取修复版本
