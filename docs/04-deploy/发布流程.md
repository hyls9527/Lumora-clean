# 发布流程文档 — Lumora

## 发布检查清单

### 发布前

| # | 检查项 | 命令 | 要求 |
|---|--------|------|------|
| 1 | TypeScript 零报错 | `npx tsc --noEmit` | exit 0 |
| 2 | 前端测试全部通过 | `npx vitest run` | 340/340 |
| 3 | Rust 测试全部通过 | `cd src-tauri && cargo test --lib` | 57/57 |
| 4 | Rust 编译通过 | `cd src-tauri && cargo check` | exit 0 |
| 5 | 前端构建成功 | `npm run build` | exit 0 |
| 6 | 版本号已更新 | `src-tauri/tauri.conf.json` version 字段 | 与 tag 一致 |
| 7 | CHANGELOG 已更新 | `CHANGELOG.md` | 包含本次变更 |

### 发布

```bash
# 1. 确认 main 分支干净
git status

# 2. 创建 tag
git tag v0.5.2

# 3. 推送 tag 触发 Release CI
git push origin v0.5.2
```

### CI 自动执行

```
push tag v* 触发 release.yml
  │
  ├─ ubuntu-latest: 构建 AppImage + DEB
  ├─ windows-latest: 构建 MSI + NSIS EXE
  ├─ macos-latest: 构建 DMG
  │
  ├─ 创建 GitHub Release (draft)
  │   ├─ 上传所有构建产物
  │   └─ 生成 release notes
  │
  └─ 生成 latest.json
      ├─ 包含各平台下载链接
      ├─ 包含版本号和发布日期
      └─ 上传到 Release assets
```

### 发布后

| # | 检查项 | 说明 |
|---|--------|------|
| 1 | GitHub Release 已创建 | 检查 Releases 页面 |
| 2 | 构建产物已上传 | 确认 MSI/EXE/AppImage/DEB/DMG 存在 |
| 3 | latest.json 已生成 | 确认 updater 可用 |
| 4 | 安装测试 | 在 Windows 上测试 MSI 安装 |

## 版本号规范

遵循 Semver (语义化版本)：

```
v主版本.次版本.修订号

v0.5.1
│ │  └── 修订号：bug fix
│ └───── 次版本：新功能
└─────── 主版本：破坏性变更
```

当前版本：**v0.5.1**

## 回滚流程

如果发布版本有问题：

1. 在 GitHub Releases 页面删除该版本的 Release
2. 删除对应 tag：`git tag -d v0.5.2 && git push origin :refs/tags/v0.5.2`
3. 修复问题后重新发布
