# 部署架构文档 — Lumora

> **审计原则**：每个部署配置必须追溯到具体文件。

## 1. 构建目标

| 平台 | 架构 | 格式 | CI 环境 |
|------|------|------|---------|
| Windows | x86_64 | NSIS (.exe) | windows-latest |
| macOS | aarch64 | .dmg, .app.tar.gz | macos-latest |
| Linux | x86_64 | .AppImage, .deb | ubuntu-latest |

**代码依据**：`.github/workflows/release.yml` matrix

---

## 2. CI/CD 流程

### 2.1 持续集成 (ci.yml)

**触发条件**：
- Push to `main`
- Pull request to `main`

**流程**：

```
┌─────────────┐     ┌─────────────┐
│  Frontend   │     │    Rust     │
│  Job        │     │    Job      │
├─────────────┤     ├─────────────┤
│ npm ci      │     │ rust-toolchain│
│ tsc --noEmit│     │ cargo test  │
│ vitest run  │     │ cargo check │
│ npm run build│    │             │
└─────────────┘     └─────────────┘
```

**代码依据**：`.github/workflows/ci.yml`

---

### 2.2 发布流程 (release.yml)

**触发条件**：Push tag `v*`

**流程**：

```
┌─────────────────┐
│ 1. Verify       │
│ Version Sync    │
└────────┬────────┘
         │
┌────────▼────────┐
│ 2. Tests        │
│ Frontend + Rust │
└────────┬────────┘
         │
┌────────▼────────┐
│ 3. Build        │
│ (3 platforms)   │
│ - Ubuntu        │
│ - Windows       │
│ - macOS         │
└────────┬────────┘
         │
┌────────▼────────┐
│ 4. Release      │
│ - Create Draft  │
│ - Upload Assets │
│ - Sign (minisign)│
└────────┬────────┘
         │
┌────────▼────────┐
│ 5. Updater JSON │
│ - Generate      │
│ - Upload        │
└─────────────────┘
```

**代码依据**：`.github/workflows/release.yml`

## 3. 版本同步检查

**检查点**：三个文件的版本号必须一致

| 文件 | 字段 | 示例 |
|------|------|------|
| `Cargo.toml` | `version` | `0.6.0` |
| `package.json` | `version` | `0.6.0` |
| `tauri.conf.json` | `version` | `0.6.0` |

**代码依据**：`release.yml` verify job

## 4. 构建配置

### 4.1 前端构建

**工具**：Vite

**命令**：
```bash
npm run build
```

**输出**：`dist/`

**配置**：`vite.config.ts`

---

### 4.2 后端构建

**工具**：Cargo + Tauri CLI

**命令**：
```bash
cargo tauri build
```

**输出**：`src-tauri/target/release/`

**配置**：`src-tauri/Cargo.toml`

---

### 4.3 应用打包

**工具**：`tauri-apps/tauri-action@v0`

**配置**：`src-tauri/tauri.conf.json`

```json
{
  "bundle": {
    "active": true,
    "targets": "all",
    "icon": ["icons/32x32.png", "icons/128x128.png", ...],
    "windows": {
      "nsis": {
        "installerIcon": "icons/icon.ico",
        "languages": ["SimpChinese"],
        "installMode": "currentUser"
      }
    }
  }
}
```

## 5. 签名配置

### 5.1 签名工具

**工具**：minisign

**密钥**：
- 公钥：`tauri.conf.json` → `plugins.updater.pubkey`
- 私钥：GitHub Secrets → `TAURI_SIGNING_PRIVATE_KEY`
- 密码：GitHub Secrets → `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`

**代码依据**：`tauri.conf.json`, `release.yml`

---

### 5.2 签名流程

1. 构建应用
2. 使用 minisign 私钥签名
3. 生成 `.sig` 文件
4. 上传到 GitHub Release

---

## 7. 安全配置

### 7.1 CSP (Content Security Policy)

```json
{
  "app": {
    "security": {
      "csp": "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; img-src 'self' asset: https://asset.localhost; font-src 'self' https://fonts.gstatic.com; connect-src 'self' http://localhost:* https://localhost:* https://github.com ipc: tauri:"
    }
  }
}
```

**代码依据**：`tauri.conf.json`

---

### 7.2 权限配置

**GitHub Actions 权限**：
```yaml
permissions:
  contents: write
```

**Secrets**：
- `GITHUB_TOKEN`：自动提供
- `TAURI_SIGNING_PRIVATE_KEY`：minisign 私钥
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`：私钥密码
