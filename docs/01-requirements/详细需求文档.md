# 详细需求文档 — Lumora v0.5.1

> 基于实际代码库逆向生成，所有需求均有代码依据。

## 1. 图片管理

### US-IMG-01: 导入图片
**作为** 用户，**我希望** 通过拖拽文件/文件夹或点击按钮导入图片，**以便** 将图片纳入管理。

**验收标准：**
- 支持 png/jpg/jpeg/webp/avif/bmp/gif/tiff 格式
- 单文件和文件夹递归扫描均可用
- 重复文件（相同 file_hash）自动跳过
- 导入后提取图片尺寸、文件大小、创建时间
- PNG 文件自动解析 SD/ComfyUI 元数据（prompt、seed、model）

**代码依据：** `src-tauri/src/commands/images.rs` — `import_images`、`scan_folder`

### US-IMG-02: 浏览图片列表
**作为** 用户，**我希望** 分页浏览已导入的图片，**以便** 快速查找目标图片。

**验收标准：**
- 按导入时间倒序排列
- 支持分页（page + perPage 参数）
- 显示总数和当前页码
- 支持网格/列表视图切换

**代码依据：** `src-tauri/src/commands/images.rs` — `list_images`

### US-IMG-03: 搜索图片
**作为** 用户，**我希望** 通过关键词搜索图片文件名和元数据，**以便** 快速定位图片。

**验收标准：**
- FTS5 全文搜索，匹配 file_path 和 metadata_json
- 特殊字符自动转义（防止 FTS5 语法错误）
- 结果按相关度排序，上限 200 条
- 支持字段范围搜索（prompt/seed/model/negative_prompt/sampler）

**代码依据：** `src-tauri/src/commands/images.rs` — `search_images`、`search_images_advanced`

### US-IMG-04: 评分与收藏
**作为** 用户，**我希望** 对图片评分（0-5）和收藏，**以便** 标记重要图片。

**验收标准：**
- 评分使用梅花印样式，0-5 分
- 收藏使用藏书印 ◆ 样式
- 支持收藏列表独立查看
- 评分和收藏操作支持乐观更新 + 失败回滚

**代码依据：** `src-tauri/src/commands/images.rs` — `update_rating`、`toggle_favorite`、`list_favorites`

### US-IMG-05: 变体组
**作为** 用户，**我希望** 相同 prompt 的不同 seed 图片自动分组，**以便** 对比变体效果。

**验收标准：**
- 导入时根据 metadata_json.prompt 自动创建/关联变体组
- 不同 prompt 分配不同组
- 可查看变体组内所有图片

**代码依据：** `src-tauri/src/commands/images.rs` — `get_variant_group_images`、`find_or_create_variant_group`

## 2. 标签系统

### US-TAG-01: 管理标签
**作为** 用户，**我希望** 创建、编辑、删除标签，**以便** 分类管理图片。

**验收标准：**
- 标签名唯一
- 支持自定义颜色
- 可修改标签名和颜色
- 删除标签时自动清理关联关系

**代码依据：** `src-tauri/src/commands/tags.rs`

### US-TAG-02: 图片标签
**作为** 用户，**我希望** 给图片添加/移除标签，**以便** 按标签筛选。

**验收标准：**
- 一张图片可添加多个标签
- 一个标签可关联多张图片
- 支持批量添加/移除标签

**代码依据：** `src-tauri/src/commands/tags.rs` — `add_tag_to_image`、`remove_tag_from_image`

## 3. 回收站

### US-TRASH-01: 软删除与恢复
**作为** 用户，**我希望** 删除图片后可在回收站恢复，**以便** 防止误删。

**验收标准：**
- 软删除设置 deleted=1 和 deleted_at 时间戳
- 回收站列表按删除时间倒序
- 恢复操作重置 deleted=0
- 永久删除级联清理：image_tags、embeddings、vec_embeddings、analysis_history
- 支持批量操作（批量删除、恢复、永久删除）

**代码依据：** `src-tauri/src/commands/trash.rs`

## 4. AI 分析

### US-AI-01: 图片分析
**作为** 用户，**我希望** 使用 AI 模型分析图片内容，**以便** 自动生成描述和标签。

**验收标准：**
- 调用 Ollama llava 模型分析图片
- 分析结果存储到 analysis_history 表
- 可查看历史分析记录
- AI 分析有 5 秒超时保护
- 支持自动将分析结果转为标签

**代码依据：** `src-tauri/src/commands/ai.rs`

### US-AI-02: 语义搜索
**作为** 用户，**我希望** 用自然语言描述查找图片，**以便** 快速找到目标图片。

**验收标准：**
- 文本通过 nomic-embed-text 模型转为向量
- 向量通过 sqlite-vec 进行相似度搜索
- 结果按相似度排序，附带分数
- 搜索结果有 LRU 缓存（30 分钟 TTL，5MB 上限）
- 支持图片到图片的视觉相似搜索（CLIP 嵌入）

**代码依据：** `src-tauri/src/commands/embeddings.rs`、`src/lib/api/semantic.ts`、`src/lib/api/semanticCache.ts`

## 5. 导出

### US-EXP-01: 导出图片
**作为** 用户，**我希望** 将选中图片导出到指定目录，**以便** 备份或分享。

**验收标准：**
- 支持选择导出格式
- 支持自定义文件名模板（renameTemplate）
- 批量导出选中图片

**代码依据：** `src-tauri/src/commands/export.rs`、`src/lib/api/images.ts`

### US-EXP-02: 数据库备份
**作为** 用户，**我希望** 导出/导入整个数据库，**以便** 迁移或备份数据。

**验收标准：**
- 导出 SQLite 数据库文件到指定路径
- 导入时先写入暂存文件，避免破坏活跃 WAL
- 导入后需要重启应用

**代码依据：** `src-tauri/src/commands/backup.rs`

## 6. 设置

### US-SET-01: 语言与主题
**作为** 用户，**我希望** 切换中英文界面和明暗主题。

**验收标准：**
- 支持中文/英文切换（i18n）
- 设置持久化到 Rust 后端（get_setting/set_setting）
- 主题切换即时生效

**代码依据：** `src-tauri/src/commands/settings.rs`、`src/lib/i18n.ts`

## 7. 系统集成

### US-SYS-01: Ollama 状态检测
**作为** 用户，**我希望** 在侧边栏看到 Ollama 连接状态，**以便** 知道 AI 功能是否可用。

**验收标准：**
- 每 60 秒轮询 Ollama 状态
- 显示连接成功/失败状态
- Ollama 不可用时 AI 功能降级（不崩溃）

**代码依据：** `src/hooks/useOllamaStatus.ts`、`src-tauri/src/ollama.rs`

### US-SYS-02: 自动更新
**作为** 用户，**我希望** 应用自动检测新版本并提示更新。

**验收标准：**
- 启动时检查 GitHub Releases
- 有新版本时在侧边栏显示更新提示
- 支持一键下载并安装
- 当前版本 updater JSON 使用空签名

**代码依据：** `src/hooks/useUpdater.ts`、`src-tauri/tauri.conf.json` plugins.updater

### US-SYS-03: 快捷键
**作为** 用户，**我希望** 使用键盘快捷键操作应用。

**验收标准：**
- ⌘K 打开命令面板
- 方向键导航图片列表
- 大部分操作支持键盘（侧边栏导航除外）

**代码依据：** `src/hooks/useKeyboardNav.ts`、`src/components/ui/CommandPalette.tsx`
