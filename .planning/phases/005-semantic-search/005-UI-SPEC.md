## Interaction Contracts

### SEM-01: Natural Language Search Input

**Location:** GalleryPage toolbar area. Inserted between the existing toolbar (sort buttons) and the BatchEmbeddingBar, above the TagFilterBar. Positioned as a full-width bar with left-aligned input and right-aligned mode toggle.

**Layout structure (top to bottom in gallery):**
```
[Section heading: "Gallery"]
[Toolbar: sort buttons ...................... select all / list view]
[BatchEmbeddingBar (conditional)]
[NEW: SemanticSearchBar — input + mode toggle]  <-- Phase 005 insertion
[TagFilterBar]
[Image grid / masonry]
```

**Input behavior:**
- Placeholder text: "Describe what you're looking for..."
- Accepts free-form natural language: "sunset over mountains", "portrait with warm lighting", "minimalist architecture"
- Debounce: 300ms before triggering search (slightly longer than CommandPalette's 150ms, reflecting semantic search's heavier feel)
- Minimum query length: 2 characters before triggering suggestions
- Clear button (text "Clear" / "清除") appears when query is non-empty, right side of input
- Keyboard: Escape clears query, Enter triggers immediate search

**Input styling:**
- Height: 36px (h-9)
- Background: `bg-surface`
- Border: `border border-border`
- Focus: `border-accent ring-1 ring-accent/20`
- Text: `text-[14px] font-serif text-text`
- Placeholder: `text-text-muted`
- Border radius: `rounded-[4px]`
- Left internal padding: 12px
- Transitions: 200ms ease-out on border-color and box-shadow

### SEM-02: Similarity Scores

**Display location:** ImageCard, bottom-left region. Positioned adjacent to the existing format badge. Appears only when semantic search mode is active and a search has been performed.

**Badge styling (SimilarityScore component):**
- Format: "Match: 87%" (EN) / "匹配: 87%" (ZH)
- Font: JetBrains Mono 11px, tabular-nums for aligned digits
- Border radius: 4px (matches badge component)
- Padding: 2px 6px (compact, non-distracting)

**Score color coding:**

| Score Range | Text Color | Background | Border |
|-------------|-----------|------------|--------|
| 80-100% | `text-accent` | `bg-accent-subtle` | `border-accent/20` |
| 50-79% | `text-text-secondary` | transparent | `border-border` |
| 0-49% | `text-text-muted` | transparent | `border-border-subtle` |

**Interaction:**
- Badge is always visible in semantic search mode (not hover-revealed -- the score is the primary value proposition of semantic search)
- Hover on badge shows Tooltip: "Semantic similarity score -- higher means better match" / "语义相似度分数 — 越高越匹配"
- No click action -- score is informational
- Keyboard: not focusable

### SEM-03: Autocomplete / Search Suggestions

**Location:** Dropdown panel directly below the search input. Appears when user has typed 2+ characters. Dismissed on Escape, on blur (with 150ms delay to allow click), or when a suggestion is selected.

**Dropdown styling:**
- Width: matches search input width
- Background: `bg-surface`
- Border: `border border-border`
- Border radius: `rounded-[6px]` (matches CommandPalette panel)
- Shadow: `shadow-elevated`
- Max height: 240px, overflow-y auto
- Z-index: 40 (above gallery content, below CommandPalette's 100)

**Suggestions structure:**
```
[Section heading: "SUGGESTIONS" / "建议"]  -- 10px Label, uppercase, text-faint
[suggestion item 1]                        -- 12px Small, text-secondary
[suggestion item 2]
[suggestion item 3]
...
[divider: border-subtle]
[Section heading: "TRY DESCRIBING" / "试试描述"]  -- for guided discovery
["sunset over mountains"                  ]
["portrait with warm lighting"            ]
```

**Suggestion item states:**
- Idle: DM Sans 12px, text-secondary, transparent background, 4px radius
- Hover: text-text, bg-accent-subtle, 200ms ease-out
- Keyboard-focused: bg-accent-subtle, text-text (matches CommandPalette pattern)

**Suggestion selection:**
- Click: populates input, triggers search immediately
- Keyboard: Arrow Up/Down to navigate, Enter to select
- Selected suggestion fills the input and search executes

**Mock suggestions (10-15 common descriptive terms):**
English: "sunset", "mountains", "portrait", "landscape", "architecture", "night", "water", "forest", "urban", "minimalist", "warm lighting", "golden hour", "abstract", "nature", "cityscape"

### SEM-04: Empty State

**Context 1: No query entered (idle state)**

When the search input is empty and mode is "Semantic":
- Gallery displays all images normally (no filtering)
- Search bar shows placeholder: "Describe what you're looking for..."
- No autocomplete dropdown
- No similarity scores on cards

**Context 2: Query entered, no results**

When a semantic search query returns zero results:
- Gallery area shows a themed empty state (replacing the image grid)
- Follows the existing gallery empty state pattern:

```
[centered in gallery area]

    ┌──────────┐
    │    ♢     │    ← 48x48 container, 2px border-radius,
    └──────────┘      bg-bg, border border-border-subtle,
                      font-serif 24px text-text-faint

  No Results Found      ← Heading, Noto Serif SC 18px, text-text

  Try a different description                            ← Body, DM Sans 14px, text-muted
  "sunset over mountains" or "portrait with warm lighting"

         [Browse Gallery]  ← Button, 12px, accent link style
                               (clears search, switches to Exact mode)
```

**Context 3: Semantic search mode active, no query, gallery has no images**
- Falls through to existing gallery empty state (import prompt)
- Not a Phase 005-specific concern

### SEM-05: Mode Toggle

**Location:** Right side of the SemanticSearchBar component.

**Visual design:**
- Two text segments: "Exact" / "精准" and "Semantic" / "语义"
- Style follows existing sort button pattern in gallery toolbar -- underline-based active indicator
- No container background, no border around the toggle group

**HTML structure:**
```html
<div class="flex items-center gap-1">
  <button class="px-2 pb-2.5 mb-[-1px] font-serif text-[12px] border-b-2
                  [active:border-accent text-text] [inactive:border-transparent text-text-muted]
                  transition-all duration-200 ease-out">
    Exact
  </button>
  <button class="px-2 pb-2.5 mb-[-1px] font-serif text-[12px] border-b-2
                  [active:border-accent text-text] [inactive:border-transparent text-text-muted]
                  transition-all duration-200 ease-out">
    Semantic
  </button>
</div>
```

**Behavior:**
- Default mode: "Exact" (preserves backward compatibility -- gallery works as before)
- Clicking a segment switches the mode
- Mode switch clears current search results
- Mode switch preserves the search query text (so user can toggle and compare)
- Keyboard shortcut hint displayed as subtle text: "Exact (⌘K)" and "Semantic (⌘⇧K)"

**Mode behaviors:**

| Aspect | Exact Mode | Semantic Mode |
|--------|-----------|---------------|
| Search trigger | ⌘K opens CommandPalette | Typing in SemanticSearchBar input |
| Filter method | Tag/path substring match (existing `getFilteredImages`) | Mock semantic similarity ranking |
| Results display | Gallery grid filtered by match | Gallery grid sorted by score, with SimilarityScore badges |
| Autocomplete | CommandPalette: command matching | SemanticSearchBar: descriptive term suggestions |
| Store used | `app-store.searchQuery` | `semantic-search-store.query` + `semantic-search-store.results` |
| Disabled embedding note | not applicable | If no images have embeddings, show subtle note: "Embeddings needed" |

**Keyboard shortcuts:**
- `⌘K`: Opens CommandPalette (always, regardless of mode)
- `⌘⇧K`: Focuses SemanticSearchBar input and switches to Semantic mode (new shortcut)
- `Escape`: Clears search query in both modes; returns focus to gallery

## Error States

| Context | Treatment |
|---------|-----------|
| Semantic search API stub throws/fails | Input remains interactive. Gallery area shows: "Search unavailable -- try exact match instead" / "搜索不可用 -- 请尝试精确匹配" in text-danger, 14px DM Sans, centered. Below it, a "Switch to Exact" button (accent link style, 12px). Toast notification NOT used -- error is contextual to the search bar area. |
| Autocomplete API stub fails | Dropdown shows "Suggestions unavailable" / "建议不可用" in text-faint, 12px. Input typing still works. Search still works without suggestions. Degrade gracefully. |
