### Accent Reserved For

The `--color-accent` (#7a5c12) is reserved for exactly these elements -- never applied to decorative or non-interactive surfaces:

1. Primary button backgrounds (variant: default) and their hover states
2. Active navigation state -- left bar + bold text in sidebar
3. Selection indicators -- checkbox fill on ImageCard, ring on focused card
4. Status bars -- embedding progress bar (background fill), format distribution bars
5. Favorite/bookmark stamp (◆) background
6. Link text and underline decoration
7. Focus-visible ring (1px, opacity 0.3)
8. Embedding status "embedded" indicator -- accent-subtle background + accent border

## Interaction Contracts

### EMB-01: ImageCard Embedding Status Indicator

**Location:** ImageCard, bottom-left region, adjacent to format badge. Only visible on hover (following existing pattern of hover-revealed UI).

**States:**
1. **Embedded** -- small accent-toned dot or "✓" text marker, no interaction needed
2. **Pending** -- muted circle "○", non-interactive
3. **Error** -- subtle "✗" in danger color, hover shows tooltip with error detail

**Interaction:**
- Hover over status marker shows Tooltip with status text and (if embedded) generation timestamp
- No click action -- status is informational only
- Keyboard: not focusable (decorative/informational)

**Pattern:** Follows existing format badge in ImageCard -- small, text-only, non-distracting, reveal on hover.

### EMB-02: DetailPanel Embedding Section

**Location:** DetailPanel scroll area. Insert as a new section between Analysis and Score (or after Score if no analysis data). Uses existing SectionLabel + InfoRow pattern.

**Content when embedded:**
```
SectionLabel: "EMBEDDING" (10px uppercase)
InfoRow: "Dimensions" | "1536" (mono)
InfoRow: "Model" | "clip-vit-base-patch32"
InfoRow: "Generated" | "<date>" 
```

**Content when pending:**
```
SectionLabel: "EMBEDDING"
Text: "Not yet generated" (12px, text-muted)
```

**Content when error:**
```
SectionLabel: "EMBEDDING"
Text: "Generation failed" (12px, text-danger)
Button: "Retry" (12px, accent link style)
```

### EMB-03: Dashboard Embedding Stats

**Location:** Dashboard. Insert as a directory-style stat row in the existing stat block (between "Rated" and the dotted separator before "Rating Distribution").

**Format:**
```
[Label left] ...................... [Value right]
EMBEDDED          ··············  142 / 200
```

**Pattern:** Matches existing directory stats exactly -- 10px uppercase label, dotted separator, 15px font-medium value, 2px border-dotted bottom separator.

Empty state: stat row shows "0 / 200" with faint/muted styling.

### EMB-04: Batch Embedding Generation

**Trigger:** Appears as a contextual action bar when `selectedIds.size >= 1` in the Gallery. Positioned above the image grid or as a toolbar strip.

**States:**
1. **Idle (selection present):** Shows selection count + "Generate Embeddings" button
2. **Generating:** Button replaced with progress bar + item count (e.g., "34 of 142") + "Cancel" text button
3. **Complete:** Progress bar fills, brief "Complete" toast (3s auto-dismiss, success variant)
4. **Error:** Toast with "Embedding generation failed -- Try again" (warning variant, 5s dismiss)

**Interaction:**
- Click "Generate Embeddings" triggers `generateEmbeddings` API stub
- Progress updates via Zustand store polling pattern or interval-based mock
- "Cancel" is a simple interrupt -- no confirmation dialog, calls `cancelEmbeddingGeneration` API stub, resets to idle
- During generation, image selection is locked (can't deselect images being processed)

**Pattern:** Follows existing DropZone/Export patterns for progress feedback. Toast notification system already exists -- reuse directly.
