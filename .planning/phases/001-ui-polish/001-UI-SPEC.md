## Spacing Scale

Per DESIGN.md. All values are multiples of 4. Includes 12px and 20px as intermediate stops on the 4px grid.

| Token | Value | Usage |
|-------|-------|-------|
| 4px | 0.25rem | Icon gaps, inline padding, separator margins |
| 8px | 0.5rem | Compact element spacing, label-to-value gaps |
| 12px | 0.75rem | Card inner padding (x-axis), tight section gaps |
| 16px | 1rem | Default element spacing, card padding (y-axis) |
| 20px | 1.25rem | Medium section padding, panel inner spacing |
| 24px | 1.5rem | Section padding, dialog inner padding |
| 32px | 2rem | Layout gaps, page-level content separation |
| 48px | 3rem | Major section breaks |
| 64px | 4rem | Page-level spacing, hero sections |

**Exceptions for this phase:** None. When auditing, if a spacing value clearly falls outside this scale, correct to the nearest scale value per D-11.

## Color

60/30/10 split per DESIGN.md palette, implemented as CSS custom properties in `src/index.css` `@theme` block.

| Role | Value | CSS Variable | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | #f2ede4 | `--color-bg` | Page background (象牙纸页) |
| Secondary (30%) | #f7f2ea | `--color-surface` | Cards, sidebar, nav, panels, dialogs |
| Accent (10%) | #7a5c12 | `--color-accent` | See reserved-for list below |
| Destructive | #8b3030 | `--color-danger` | Delete actions, irreversible warnings only |
| Success | #4a7a3a | `--color-success` | Confirmation states, completion indicators |

**Accent reserved exclusively for:**
- Active sidebar nav item (left gold bar + bold text)
- Sort button active underline (`border-b-2` + accent)
- PlumFlower rating SVG fill (梅花印 filled petals)
- Bookmark indicator (藏书印 ◆ character)
- Link/action hover states on text
- Import progress bar fill
- Focus ring on interactive elements (subtle use)

**Accent must NOT appear on:**
- Default button backgrounds (buttons use surface/secondary tones)
- Non-active navigation items
- General decorative elements
- Background washes (use `--color-accent-subtle` rgba variant instead)

**Border colors:**
- `--color-border`: `rgba(139,115,75,0.10)` — card/panel divides
- `--color-border-subtle`: `rgba(139,115,75,0.05)` — whisper-weight separators

**Text colors:**
- `--color-text`: `#2a2118` — primary body text (研磨墨)
- `--color-text-secondary`: `#6b5d48` — secondary labels
- `--color-text-muted`: `#7a6b58` — muted descriptions
- `--color-text-faint`: `#9a8c78` — placeholder/disabled text

**Shadow system (3-layer, warm brown cast):**
- `--shadow-card`: Card default (1px border + subtle lift)
- `--shadow-card-hover`: Card hover (deeper shadow, NO scale transform per D-06)
- `--shadow-elevated`: Dialogs, dropdowns, elevated surfaces

## Copywriting Contract

Phase 001 is a visual polish phase -- no new copy is introduced. All copy must remain sourced from the existing i18n system (`src/i18n/{locale}.json`). The contract below maps user-facing states to their i18n keys.

| Element | Copy (zh) | i18n Key | Notes |
|---------|-----------|----------|-------|
| Primary CTA (import) | 拖放图片到此处导入 | `dropzone.title` | DropZone component; also "导出" on ExportDialog |
| Gallery empty state heading | 暂无图片 | `gallery.empty.title` | Shown when images array is empty |
| Gallery empty state body | 导入图片开始使用 | `gallery.empty.subtitle` | Prompts next action |
| Curation empty state | 暂无可策展内容 | `curation.empty.title` | Shown when no images queued for curation |
| Trash empty state | 回收站为空 | `trash.empty.title` | Shown when trash is empty |
| Tags empty state | 暂无标签 | `tags.empty` | Shown when no tags exist |
| General empty state | 此处尚无藏品 | `empty.no_images` | Catch-all empty state |
| Loading state | 研墨中… | `loading.grinding` / `gallery.loading` | Used during import/simulated loading |
| Error state (generic) | 此路不通 | `error.blocked` | Catch-all error from ErrorBoundary |

**Destructive actions in this phase:**

| Action | Label (zh) | Confirmation Approach |
|--------|------------|----------------------|
| Delete image from DetailPanel | 删除 | Immediate action; label implies consequence. No secondary confirmation dialog in current implementation. |
| Empty trash (TrashPage) | 清空回收站 | Direct button; the recycle bin metaphor implies reversibility (restore available). |

**No new copy strings may be added in this phase.** The polish scope is visual alignment only.

### D-06: Remove hover:scale from ImageCard.tsx

- **Remove:** `hover:scale-[1.02]` on card wrapper (line ~48)

### D-07: PlumFlower Shared Component

- **Location:** `src/components/ui/plum-flower.tsx`
- **Props interface:**
```ts
interface PlumFlowerProps {
  filled: boolean
  size?: number  // default 18 (px), used for both width and height
}
```
- **SVG geometry:** 5-petal plum flower + center circle. Petal fill uses `var(--color-accent)` when filled, `rgba(139,115,75,0.2)` stroke when empty. Center circle uses `var(--color-accent)` when filled, `rgba(139,115,75,0.12)` when empty.
- **Consumers after extraction:** DetailPanel.tsx (rating display), ImageCard.tsx (card rating)
- **Remove:** Independent PlumFlower definitions from both DetailPanel.tsx and ImageCard.tsx

### D-08/D-09: Visual Consistency Audit

**Audit checklist for each of 26 components:**

1. Colors use `--color-*` CSS custom properties or Tailwind tokens (not hardcoded hex)
2. Font families match DESIGN.md roles (Noto Serif SC for headings, DM Sans for body)
3. Spacing values from DESIGN.md scale only (4/8/12/16/20/24/32/48/64)
4. Border radii match DESIGN.md material logic: cards=2px, buttons=4px, inputs=4px, badges=4px, dialogs=6px
5. Shadows use the 3-layer system, never custom box-shadows
6. Transitions are 200ms ease-out, never 100ms, never spring/bounce

**Components to audit:**
- UI primitives (12): badge, button, card, dialog, dropdown-menu, input, scroll-area, separator, sheet, skeleton, tabs, tooltip
- Business components (10): CommandPalette, DetailPanel, DropZone, ErrorBoundary, ExportDialog, ImageCard, PageErrorBoundary, Sidebar, TagManager, VirtualizedGrid
- Pages (5): CurationPage, DashboardPage, GalleryPage, SettingsPage, TrashPage

**UI primitives rule (D-10):** Keep as-is, tweak only specific DESIGN.md deviations. They already use CVA variants, correct radii, and correct spacing. Focus audit on: color token usage (no hardcoded hex), shadow correctness, transition timing.
